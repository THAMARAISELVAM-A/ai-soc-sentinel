# Contributing to AI SOC Sentinel

Thank you for your interest in contributing to AI SOC Sentinel! We welcome contributions from everyone.

## Getting Started

1. **Fork the Repository**: Create your own copy of the project on GitHub.
2. **Clone the Repository**: Clone your fork to your local machine.
3. **Create a Branch**: Create a new branch for your changes (`git checkout -b feature/my-new-feature`).
4. **Make Changes**: Implement your changes and commit them with descriptive messages.
5. **Run Tests**: Ensure all tests pass before submitting your pull request.
6. **Submit a Pull Request**: Submit your pull request to the central repository for review.

## Code of Conduct

Help us keep AI SOC Sentinel a welcoming and inclusive community. Please follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Coding Standards

- **Use Prettier**: All code should be formatted using Prettier.
- **Add Documentation**: Use JSDoc to document your code.
- **Write Tests**: Include tests for any new features or bug fixes.
- **Follow MITRE ATT&CK**: When adding new threat signatures, please refer to the MITRE ATT&CK framework.

## Pull Request Guidelines

- **Keep PRs small**: Smaller PRs are easier to review and merge.
- **Describe your changes**: Provide a clear and concise description of what your PR does.
- **Link related issues**: If your PR fixes a bug or implements a feature, link it in the description.
- **Be responsive**: Respond to any feedback or requests for changes from the maintainers.
